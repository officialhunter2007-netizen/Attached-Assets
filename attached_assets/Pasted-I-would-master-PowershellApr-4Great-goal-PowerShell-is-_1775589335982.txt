Let me just explain this one directly since it needs a bit of background knowledge to reason through.
In PowerShell, > is actually used for redirecting output to a file — not for comparing numbers. So using it in an if condition causes unexpected behavior. PowerShell provides its own comparison operators instead:
SymbolPowerShell operatorMeaning>-gtgreater than<-ltless than==-eqequal to!=-nenot equal to
So the safe, correct way to write your login alert is:
powershell$failedAttempts = "8"

if ([int]$failedAttempts -gt 5) {
    Write-Host "ALERT: Too many failed logins!"
}
The [int] forces the string into a real number, and -gt does a proper numerical comparison. Both pieces together make it reliable.
